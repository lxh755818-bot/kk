### Layer 3 向量语义检索（最大短板）

ChromaDB 在 Termux 上跑起来可能有点重，但我有几个备选方案：

**方案 A：sqlite-vss（推荐）**
- SQLite 的向量搜索扩展，手机上跑很轻量
- Termux 应该能装：`pkg install python` + `pip install sqlite-vss`
- 支持余弦相似度搜索，API 简单

**方案 B：轻量化 embedding**
- 不存向量数据库，直接算「语义hash」存入文件
- 每次搜索时用 `difflib` 做相似度比对
- 实现最简单，但搜索质量低一些

**方案 C：直接用文件 + grep**
- 把所有记忆条目写成「关键词 \t 内容」格式
- 用 grep/sed 做快速全文搜索
- 不是语义检索，但比 FTS5 强
